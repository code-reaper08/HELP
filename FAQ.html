How to create an issue

A good issue should contain all the relevant details, so as the reviewers can easily identify the domain, your name, and the type of contribution you are going to provide.

To know how to create an issue, see here


Receiving multiple emails from GitHub in your inbox

That should be because you authored the issue so, you are by default subscribed to get notifications. You can feel free to unsubscribe from the issue notifications.

To know how to unsubscribe from an issue thread, see here


How to get the issue assigned

You need to get assigned to the issue, before you start working on it, so ask (I mean tag them in GitHub) the mentors/supervisors to assign the issue to you.

Don't work on the issue, without assignment.


Having merge conflicts

Merge conflicts happen when you merge branches that have competing commits, and Git needs your help to decide which changes to incorporate in the final merge.

You can resolve a merge conflict following the steps here
